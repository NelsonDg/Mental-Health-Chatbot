# MindCare: AI Mental Health Support Chat

A conversational web app that detects the emotional tone of what a user types (happy,
anxious, grief, stressed, and 20+ other states) and responds with an empathetic,
context-aware reply generated by Google's Gemini API. Includes voice input/output,
a hardcoded crisis-hotline safety net that never depends on the AI model, dark mode,
and a fully responsive layout.

> **Disclaimer:** This is a portfolio project, not a clinical tool. It is not
> reviewed by mental health professionals, not private/HIPAA-compliant, and not a
> substitute for real crisis support. If you or someone you know is in crisis, please
> contact a real crisis line (e.g. 988 in the US/Canada, or your local emergency number).

## Features

- **Emotion detection:** VADER sentiment analysis + keyword scoring + spaCy noun-phrase
  extraction classify each message into one of 20+ emotional states.
- **Gemini-generated responses:** replies are generated live from the detected emotion,
  sentiment score, and recent conversation history, not picked from a canned list.
- **Crisis safety net:** messages matching an expanded set of crisis phrases (with
  idiom exceptions to reduce false positives, e.g. "cut myself some slack") always
  return a hardcoded hotline response, regardless of whether the Gemini call succeeds.
- **Voice in/out:** Web Speech API for dictation and text-to-speech playback.
- **Dark mode:** follows the OS/browser `prefers-color-scheme` setting.
- **Responsive:** full-bleed layout that works down to phone-sized viewports.

## Tech stack

- **Backend:** Flask, Flask-SQLAlchemy (SQLite), NLTK (VADER), spaCy, scikit-learn, Google Gemini API (`google-genai`)
- **Frontend:** vanilla HTML/CSS/JS (no framework), Web Speech API
- **Tests:** `unittest`
- **Deployment:** Gunicorn (via Procfile)

## Setup

1. Clone the repo and create a virtual environment:
   ```bash
   python -m venv venv
   venv\Scripts\activate      # Windows
   source venv/bin/activate   # macOS/Linux
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   python -m spacy download en_core_web_sm
   python download_nltk.py
   ```

3. Copy `.env.example` to `.env` and add your [Gemini API key](https://aistudio.google.com)
   (free tier available):
   ```bash
   cp .env.example .env
   ```
   Leave `FLASK_SECRET_KEY` blank. It's generated automatically on first run and
   persisted back into `.env`.

4. Run it:
   ```bash
   python app.py
   ```
   Visit `http://127.0.0.1:5000`. Set `FLASK_DEBUG=1` in `.env` for the Flask
   auto-reloader during development.

## Running tests

```bash
python -m unittest discover tests -v
```

Covers emotion classification and crisis-phrase detection (including that idiom
exceptions like "cut myself some slack" don't false-positive).

## Deployment

A `Procfile` is included for Render/Railway/Heroku-style hosts:
```
web: gunicorn app:app
```
Set `GEMINI_API_KEY` (and optionally `FLASK_SECRET_KEY`, `FLASK_DEBUG`) as
environment variables in your host's dashboard. Never commit `.env`.
